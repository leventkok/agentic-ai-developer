# TASK 1 

Story 1:
As a coffee drinker
I want to pre-order my coffee before I arrive
So that I can skip the morning queue

Story 2:
As a coffee drinker
I want to get my coffee in the morning without waiting in a long queue
So that I don't waste my time

Story 3:
As a barista
I want to manage orders through an online system during busy hours
So that I can hand out orders without mixing up customers

Story 4:
As a coffee shop owner
I want to send my promotions to customers through the app
So that I can compete with big coffee chains and bring more people to my shop

Story 5:
As an admin
I want to add new local coffee shops to the app
So that we can grow our market share more easily

# TASK 2 

Given I am logged in and viewing a shop menu
When I select a coffee and tap "Order"
Then I see an order confirmation with estimated pickup time

- Barista can see new orders in a list
- Order shows customer name, drink, and size
- Barista can mark an order as "Ready"
- Shop receives order within 30 seconds


# TASK 3

Epic: Customer can pre-order and pay for coffee before arriving

1. customer can search for a coffee shop by location
2. customer can favorite a shop and reorder their usual coffee in one tap
3. customer can schedule the pickup time (e.g. "ready at 8:30")
4. customer can pay with a saved card or Apple Pay
5. customer can track the order status live (preparing → ready)


# TASK 4

Priority 1: Customer can view the shop menu — everything depends on this; you can't order without seeing the menu first.

Priority 2: Customer can add a coffee to the cart and select size — needed before payment; the user must choose what to buy.

Priority 3: Customer can pay through the app — high user value and high risk (money), and it must work before the order is sent.

Priority 4: Shop receives the order notification — depends on a successful payment; without this, orders never reach the shop.

Priority 5: Customer sees "ready for pickup" status — nice value, but it only matters after all the steps above work.

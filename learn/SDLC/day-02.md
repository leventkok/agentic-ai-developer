# TASK 1

1- Requirements
Input: drivers can't see passengers' stops clearly
Output: As a driver, I want to see passengers stops on a map
2- Design
Input: approved requirements (the driver wants to see stops on a map)
Output: a wireframe and API spec that show how the map screen will look
3- Implementation
Input: approved design (the map screen wireframe)
Output: working code for the map feature in a repo (PRs)
4- Testing
Input: the built map feature
Output: test results and bug reports before release
5- Deployment
Input: tested and approved build
Output: the map feature is live and drivers can use it
6- Maintenance
Input: the running app in production
Output: bug fixes, updates, and monitoring alerts


# TASK 2
Design is done when the API fields and the main logic are fully designed, so developers can start coding without guessing.

Testing is done when all tests are complete and there are no bugs or security holes, so the release is approved.


# TASK 3

The PM/PO collects all the requirements, does competitor analysis, and makes the main decisions about what the team builds.
The Designer is responsible for the UI/UX and the design of the API fields.
The Developer builds the approved design by writing the actual code.
The QA runs the tests on the application both before and after release to catch bugs.
DevOps is responsible for deploying the software and keeping it running after release.

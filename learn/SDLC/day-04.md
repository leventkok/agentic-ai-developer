 # TASK 1

Product: RotaCoffee — a mobile app to order coffee from local coffee shops.

Model: Hybrid

Why (5 bullets):

The payment system is built with Waterfall because it involves money and needs careful planning and low risk 
User data and account security are planned upfront with Waterfall
The UI is developed with Agile so the team can change the design quickly and stay different from competitors
Features are delivered in short Agile sprints to meet a fast timeline and ship quickly
Agile sprints allow frequent user feedback so the team can improve the ordering experience fast

# TASK 4

You know how apps get built and updated as developers, we usually follow something called SDLC, the Software Development Life Cycle its basically a step by step process that makes an app much easier to manage it has a few main stages, First, requirements, we figure out what the product needs and why, design, where we plan how the app will look and how it will work. implementation, which is the part where we actually write the working code and share it with the team. testing, where we check the app for bugs. deployment, when the finished version finally goes out to users. last, maintenance, we keep fixing bugs and improving the app over time.

for example, imagine a local coffee shop order app. firstly we talk to coffee shops to see if it's really needed, then we design the order screen, write the code, test it, and release it, and after launch, we keep fixing problems and adding improvements.

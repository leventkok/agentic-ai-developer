# TASK 1 

SDLC is the process that covers a software project from the beginning to the end. We use SDLC to manage the project better and to make future development easier.

# TASK 2

Requirements produces a clear understanding of what the software should do and why.
Design produces a plan for how the software will be built, such as an architecture diagram and API spec.
Implementation produces the actual working code of the software through commits and pull requests.
Testing produces confidence that the software meets requirements before users see it. (verilen örnek)
Deployment produces a released version of the software that users can access.
Maintenance produces ongoing bug fixes and improvements that keep the software running well.

# TASK 3

Instagram added a quick content sharing feature → Requirements (user analysis) + Design (UX decision) + Implementation (code) + Testing (QA before release) + Deployment (rolled out to users).


# TASK 4

If Requirements is skipped, the team may build a feature that nobody uses, which wastes time and effort.
If Design is skipped, the product can end up with a confusing interface that overwhelms users and a weak architecture.
If Testing is skipped, the software goes to users full of bugs.
